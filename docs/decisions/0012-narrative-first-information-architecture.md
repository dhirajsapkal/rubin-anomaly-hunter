# ADR-0012 — Narrative-first information architecture

**Status:** Accepted
**Date:** 2026-04-22
**Supersedes:** IA ordering portion of `docs/ux/brief.md` §3.3

## Context

After the Mission-Control Modern visual rewrite (ADR-0011), the user tested the dashboard against the Candidate Detail flow and reported:

> "The UX and IA needs work. When I look at a potential watch-list item, I don't understand what all the metrics mean. I want to know what's weird about it and speculation on what it could be."

The page was leading with null-hypothesis tests followed by orbit-fit block (a, e, i, q, Q, Marsden A1/A2/A3, σ, rms) — technically correct but opaque to a hobbyist astronomer. The original UX brief §3.3 put the null-test panel and orbit fit up top, then cutouts, light curve, orbit, cross-broker context, notes, decision bar.

## Decision

Reorganize the Candidate Detail page so the reading order is **narrative → reasoning → evidence → data → media → decision**:

1. **"WHAT'S WEIRD ABOUT THIS"** — plain-language narrative panel with an amber-accented display headline, two paragraphs explaining the trigger in prose, and a checklist of the specific evidence that satisfied each gate.
2. **"WHAT IT COULD BE"** — ranked hypothesis cards (LEADING / PLAUSIBLE / UNLIKELY / SYSTEMATIC), each with a tagline, description, "supports" bullets drawn from the current data, and "would confirm / would refute" bullets explaining what follow-up would settle the question.
3. **Evidence & data** — null-hypothesis checklist + orbit fit block (now rendered as a three-column grid: symbol, value, plain-language gloss) + cutouts strip.
4. Light curve + orbit plot (side-by-side paper cards).
5. Cross-broker context + MPC cross-match + notes.
6. Decision action bar.

Narrative content is generated by `dashboard/lib/narrative.py` — no new DB columns; the narrative is derived at render time from existing entry fields (kind, orbit fit, non-grav magnitudes, null-test results, MPC cross-match).

Two supplementary surfaces echo the same pattern:

- **Watch List rows** gain a second line below the grid — a one-line "what's weird" summary derived from the same `generate_why_flagged()` logic. Rows are now visually distinguishable at a glance (systematic-suspicion vs. threshold-wobbler vs. ISO match).
- **Tonight home page** gains a "TONIGHT'S LEAD" strip above the hero numeric that picks the single highest-signal entry and states its leading hypothesis in prose, with a direct link. Selection rule: ISO if present, else dark-comet with largest |A1|.

Every orbital symbol in the orbit fit block carries an inline plain-language gloss via `narrative.ORBITAL_ELEMENT_GLOSSARY` (e.g. `e` → *"eccentricity — 0 is a circle, <1 is bound, >1 is unbound"*).

## Consequences

- The dashboard teaches the field's vocabulary while the user triages — no separate glossary page needed.
- Hypothesis cards are generated from the entry data using deterministic rules in `narrative.py`. They are **not** ML predictions or probability estimates; they are curated hypothesis framings that the pipeline is good at producing because the category boundary cases are well-understood.
- Language invariants from ADR-0005 are preserved. The narrative never uses "candidate" before promotion and never uses "discovery" at all.
- Future entries added to categories beyond `dark_comet` and `iso` require new branches in `generate_why_flagged()` / `generate_hypotheses()`. For unhandled categories, a neutral fallback narrative renders.
- `docs/ux/brief.md` §3.3's content ordering is updated de facto. The persona, JTBD ranking (§2), empty-state design (§4), language rules (§5), decision interactions (§6), and non-goals (§7) remain authoritative.

## Alternatives considered

- **Add a glossary page.** Rejected — the user would not go look it up mid-review; annotations in-flow are cheaper to read.
- **ML-ranked hypotheses with confidence percentages.** Rejected — the demo has no training data; probability numbers would be fabricated. Discrete LEADING/PLAUSIBLE/UNLIKELY pills communicate the same useful signal without pretending to statistical rigor.
- **Put the narrative in a tooltip/hover.** Rejected — narrative is the main thing the user opens the page to read; hiding it behind interaction inverts the IA problem.
- **Generate narrative from an LLM at render time.** Rejected — adds latency, non-determinism, and a dependency for what is a finite branching problem solvable with rule-based logic.
